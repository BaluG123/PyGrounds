# PDF Notes Content Guide

Use this when adding PDF URLs before release.

## PDF Requirements

Each PDF should be:

- Publicly reachable by HTTPS.
- Mobile-readable on a phone screen.
- Smaller than necessary for fast loading.
- Matched to the exact course topic.
- Free of copyrighted material you do not have rights to distribute.
- Free of placeholder pages.
- Tested on Android release build.

## Recommended PDF Structure

For each course PDF:

1. Title page.
2. What students will learn.
3. Key concepts.
4. Worked examples.
5. Common mistakes.
6. Practice problems.
7. Answer key or hints.
8. Next steps.

## Course PDF Checklist

Create or link notes for:

- Python Basics
- Python Advanced
- NumPy
- Pandas
- Matplotlib
- Math for AI
- Linear Algebra
- Scikit-Learn
- Deep Learning
- AI Projects
- Problem Solving

## URL Testing Checklist

For every PDF URL:

- Opens in a normal browser.
- Opens on Android device.
- Does not require login.
- Does not redirect to a blocked page.
- Loads on mobile data.
- File name and title are readable.
- Content matches the course.

## Content Quality Rules

- Prefer short, focused PDFs over huge textbooks.
- Use examples that students can try in the Playground.
- Add diagrams where concepts are visual.
- Add Indian student-friendly examples where useful: marks, attendance, cricket scores, expenses, weather, exam results.
- Keep explanations respectful and beginner-friendly.

## Release Risk

Bad PDF links create a broken app experience. Do not publish until every visible PDF link is tested.
